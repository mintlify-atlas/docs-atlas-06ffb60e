## Overview

The MKing Admin API uses JWT (JSON Web Token) authentication. All authenticated endpoints require a valid token in the Authorization header.

### Authentication Flow

1. Call the `/login` endpoint with credentials
2. Receive a JWT token in the response
3. Include the token in subsequent requests using the `Authorization: Bearer <token>` header
4. Token is stored in localStorage and automatically attached by axios interceptors

## Login

<RequestExample>
```bash POST /login
curl -X POST "${VITE_BASE_URL}/login" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "your_password"
  }'
```
</RequestExample>

Authenticates a user and returns a JWT token.

### Request Body

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="password" type="string" required>
  User's password
</ParamField>

### Response

<ResponseField name="token" type="string">
  JWT authentication token to be used in subsequent requests
</ResponseField>

<ResponseField name="user" type="object">
  User information object
  
  <ResponseField name="id" type="number">
    User's unique identifier
  </ResponseField>
  
  <ResponseField name="email" type="string">
    User's email address
  </ResponseField>
  
  <ResponseField name="name" type="string">
    User's full name
  </ResponseField>
  
  <ResponseField name="role" type="object">
    User's role information
  </ResponseField>
</ResponseField>

<ResponseExample>
```json Response
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "name": "John Doe",
    "role": {
      "id": 2,
      "name": "Admin"
    }
  }
}
```
</ResponseExample>

## Get Current User

<RequestExample>
```bash GET /user
curl -X GET "${VITE_BASE_URL}/user" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

Retrieves information about the currently authenticated user.

**Authentication Required:** Yes

### Response

<ResponseField name="id" type="number">
  User's unique identifier
</ResponseField>

<ResponseField name="email" type="string">
  User's email address
</ResponseField>

<ResponseField name="name" type="string">
  User's full name
</ResponseField>

<ResponseField name="employee_id" type="number">
  Associated employee ID if applicable
</ResponseField>

<ResponseField name="role" type="object">
  User's role with permissions
</ResponseField>

<ResponseExample>
```json Response
{
  "id": 1,
  "email": "user@example.com",
  "name": "John Doe",
  "employee_id": 5,
  "role": {
    "id": 2,
    "name": "Admin",
    "permissions": [
      "products.view",
      "products.create",
      "products.update"
    ]
  }
}
```
</ResponseExample>
