## Products

### List All Products

<RequestExample>
```bash GET /product
curl -X GET "${VITE_BASE_URL}/product" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

Retrieves a list of all products.

**Authentication Required:** Yes

### Response

<ResponseField name="data" type="array">
  Array of product objects
  
  <ResponseField name="id" type="number">
    Product unique identifier
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Product name
  </ResponseField>
  
  <ResponseField name="description" type="string">
    Product description
  </ResponseField>
  
  <ResponseField name="price" type="number">
    Product price
  </ResponseField>
  
  <ResponseField name="category_id" type="number">
    Associated category ID
  </ResponseField>
  
  <ResponseField name="images" type="array">
    Array of product images
  </ResponseField>
  
  <ResponseField name="colors" type="array">
    Available colors for the product
  </ResponseField>
  
  <ResponseField name="sizes" type="array">
    Available sizes for the product
  </ResponseField>
</ResponseField>

<ResponseExample>
```json Response
{
  "data": [
    {
      "id": 1,
      "name": "T-Shirt Premium",
      "description": "High quality cotton t-shirt",
      "price": 299.99,
      "category_id": 1,
      "images": [
        {
          "id": 1,
          "url": "https://example.com/image.jpg",
          "is_primary": true
        }
      ],
      "colors": ["Red", "Blue", "Green"],
      "sizes": ["S", "M", "L", "XL"]
    }
  ]
}
```
</ResponseExample>

### Create Product

<RequestExample>
```bash POST /product
curl -X POST "${VITE_BASE_URL}/product" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: multipart/form-data" \
  -F "name=New Product" \
  -F "description=Product description" \
  -F "price=499.99" \
  -F "category_id=1" \
  -F "images[]=@/path/to/image1.jpg" \
  -F "images[]=@/path/to/image2.jpg"
```
</RequestExample>

Creates a new product with images.

**Authentication Required:** Yes  
**Content-Type:** `multipart/form-data`

### Request Body

<ParamField body="name" type="string" required>
  Product name
</ParamField>

<ParamField body="description" type="string">
  Product description
</ParamField>

<ParamField body="price" type="number" required>
  Product price
</ParamField>

<ParamField body="category_id" type="number" required>
  Category ID
</ParamField>

<ParamField body="images" type="file[]">
  Array of image files
</ParamField>

<ParamField body="colors" type="array">
  Array of color IDs
</ParamField>

<ParamField body="sizes" type="array">
  Array of size IDs
</ParamField>

### Response

<ResponseField name="id" type="number">
  Created product ID
</ResponseField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

### Update Product

<RequestExample>
```bash PUT /product/{id}
curl -X PUT "${VITE_BASE_URL}/product/1" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: multipart/form-data" \
  -F "name=Updated Product" \
  -F "price=599.99"
```
</RequestExample>

Updates an existing product.

**Authentication Required:** Yes  
**Content-Type:** `multipart/form-data`

### Path Parameters

<ParamField path="id" type="number" required>
  Product ID to update
</ParamField>

### Request Body

Same fields as Create Product (all optional for updates)
